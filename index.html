SYSTEM OVERVIEW

How HIVnext organises program data.

The system brings data collection, reporting, access control, and dashboards into a shared environment. Configuration can reflect the indicators, roles, and reporting structures used in a program.

Program and reporting configuration

Indicators, population categories, languages, reporting periods, and access permissions can be configured to reflect the operating context and data-governance requirements.

  • ▸ National indicator alignment
  • ▸ Multi-language interface
  • ▸ Role-based access (RBAC + ABAC)

Modular data workflows

Functions can be organised into separate modules for outreach, testing, prevention, case management, self-test distribution, and community-based care.

  • ▸ Outreach & screening module
  • ▸ Linkage-to-care tracking
  • ▸ Self-test & PrEP modules

Dashboards and structured exports

Authorised users can review activity and performance against targets by site, worker, population group, or reporting period as data becomes available.

  • ▸ Target and activity views
  • ▸ Site & worker-level views
  • ▸ Structured data exports

SYSTEM SETUP

A typical setup process.

STEP 01

Workflow mapping

Existing forms, indicators, user roles, reporting lines, and data flows are documented with the relevant program and M&E teams.

STEP 02

Configuration and data preparation

Fields, modules, and permissions are configured. Historical data may be cleaned, validated, and prepared for migration where appropriate.

STEP 03

User testing and orientation

Managers and field workers test the workflows, provide feedback, and receive guidance on data entry, review, and dashboard use.

STEP 04

Phased use and review

Use can begin in phases, followed by review of data quality, workflow issues, and any configuration changes that may be needed.

QUESTIONS

Frequently asked questions

What is HIVnext? +

HIVnext stands for Health Information Vault. It is a cloud-based data environment used to organise outreach, testing, linkage-to-care, prevention, reporting, and program-monitoring information for differentiated HIV services.

How is HIVnext currently used in Malaysia? +

The current Malaysia deployment includes 21 implementing partners, more than 50 service sites, and over 200 field workers. HIVnext Legacy is also reported to have reduced dashboard loading time by 80% in the existing environment.

How are indicators and reporting structures represented? +

In the current approach, indicators, population definitions, reporting periods, and data fields are documented during workflow mapping and reflected in the system configuration. Local governance and existing data systems remain part of that process.

What controls are used for confidentiality and data security? +

The current design includes role-based and attribute-based access controls (RBAC and ABAC), site- and role-specific permissions, and structured audit trails. Each implementation should also follow the applicable organisational policies, legal requirements, and data-protection procedures.

What types of organisations are represented in Malaysia? +

The Malaysia deployment includes NGOs and community-based implementing partners operating across different service-site and team sizes, with work spanning community outreach, testing, prevention, linkage to care, and program monitoring.

What does a typical setup process involve? +

A typical process includes workflow mapping, review of indicators and data fields, configuration, data preparation, user testing, orientation, and phased use. The sequence and duration vary according to program complexity, existing data, governance requirements, and available technical support.
